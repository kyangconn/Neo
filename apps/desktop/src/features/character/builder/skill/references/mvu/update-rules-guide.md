# MVU 变量更新规则

`变量更新规则.yaml` 告诉 AI 什么时候改变量、改多少、需要同步哪些派生值。状态栏变量尤其要写清触发条件，避免 AI 每回合乱改。

## 基本规则

- 自明变量可以省略 `check`，但状态栏变量必须写更新条件。
- 数值变量写 `type: number` 和 `range`。
- 同类变量用 `${生命|魔法|耐力}` 合并。
- `_` 和 `$` 前缀变量不写更新规则。
- 不写 UI，不写 HTML/CSS。

## 状态栏规则模板

```yaml
---
变量更新规则:
  主角.状态条.${生命|魔法|耐力}.value:
    type: number
    range: 0~max
    check:
      - 只有受伤、消耗、恢复、休息、战斗或显著行动后更新
      - 单次变化要符合事件强度，并在剧情结果里说明原因
      - 更新后同步对应的 Agentic Play statusBars 数值

  主角.成长.经验.value:
    type: number
    range: 0~max
    check:
      - 完成目标、战斗胜利、学习知识、推进主线或获得训练成果后增加
      - 没有明确成长收益时不增加

  主角.法术位.${环级}.剩余:
    type: number
    range: 0~上限
    check:
      - 施放对应环级法术后减少
      - 长休、补给、仪式或特殊恢复后增加
      - 每次变化后同步刷新 主角.状态条.魔法.value 为所有法术位剩余总和，max 为所有法术位上限总和

  角色.${角色名}.好感度:
    type: number
    range: 0~100
    check:
      - 根据该角色对<user>行为的感知调整
      - 普通互动通常 ±1~3，关键承诺、救援、背叛等事件可 ±5~12
      - 不在该角色不知情的情况下调整

  场景.危险度:
    type: number
    range: 0~100
    check:
      - 暴露、拖延、制造声响、战斗升级时提高
      - 成功隐蔽、解除警戒、离开危险区时降低
```

## Agentic Play 同步要求

角色卡保存后，`statusBars` 会成为 Agentic Play 新会话的初始 `gameState.player.status_bars`。剧情中状态变化时，主持人必须通过 `update_game_state` 修改同名状态条，例如：

```json
{
  "state_patch": {
    "player": {
      "status_bars": {
        "health": { "value": 72, "max": 100, "label": "生命", "description": "左臂受伤" }
      }
    }
  },
  "reason": "玩家战斗失败，承受伤害。"
}
```

## 自查清单

- [ ] 状态栏变量有明确触发条件。
- [ ] 法术位等分层资源同时更新明细和汇总条。
- [ ] 好感度只按角色实际感知更新。
- [ ] 经验值只在确实成长时增加。
- [ ] 没有把状态栏写成正文或 HTML。
