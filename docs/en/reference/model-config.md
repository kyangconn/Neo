# Model Configuration

The `ModelConfig` interface defines the connection parameters for a language model provider. Configurations are stored and managed through `settingsRepository` and the `useSettingsStore`.

## Interface

```typescript
export interface ModelConfig {
  id: string
  provider: ModelProviderType
  name: string
  baseUrl: string
  apiKey: string
  model: string
  temperature: number
  maxTokens: number
  reasoningEffort?: string
  streamingEnabled?: boolean
  createdAt: string
  updatedAt: string
}

export type ModelProviderType = 'openai-compatible'
```

## Fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `id` | `string` | — | Unique identifier (generated by `generateId()`) |
| `provider` | `ModelProviderType` | — | Provider type. Currently only `'openai-compatible'` is supported. |
| `name` | `string` | — | User-friendly label for this configuration, shown in the UI dropdown |
| `baseUrl` | `string` | — | API endpoint URL (e.g. `https://api.openai.com/v1`) |
| `apiKey` | `string` | — | API authentication key |
| `model` | `string` | — | Model identifier (e.g. `gpt-4o`, `claude-3-opus-20240229`) |
| `temperature` | `number` | `0.8` | Sampling temperature. Range: `0`–`2`. Higher values produce more random output. |
| `maxTokens` | `number` | — | Maximum number of tokens the model can generate in a single response |
| `reasoningEffort` | `string` | — | Reasoning effort level. Accepts `''` (none), `'high'`, or `'maximum'`. Used by reasoning-capable model providers. |
| `streamingEnabled` | `boolean` | — | Whether to enable token-by-token streaming of model responses. When enabled, the UI updates incrementally. |
| `createdAt` | `string` | — | ISO 8601 timestamp of creation |
| `updatedAt` | `string` | — | ISO 8601 timestamp of last update |

## Input Types

### CreateModelConfigInput

Used when creating a new model config. All fields are optional except the connection parameters:

```typescript
export interface CreateModelConfigInput {
  provider: ModelProviderType
  name: string
  baseUrl: string
  apiKey: string
  model: string
  temperature?: number
  maxTokens?: number
  reasoningEffort?: string
  streamingEnabled?: boolean
}
```

### UpdateModelConfigInput

Used when updating an existing config. All fields are optional; omitted fields keep their current values:

```typescript
export interface UpdateModelConfigInput {
  provider?: ModelProviderType
  name?: string
  baseUrl?: string
  apiKey?: string
  model?: string
  temperature?: number
  maxTokens?: number
  reasoningEffort?: string
  streamingEnabled?: boolean
}
```

## Temperature Guidelines

| Temperature | Behavior |
|-------------|----------|
| `0`–`0.3` | Deterministic, focused output — good for factual responses |
| `0.4`–`0.7` | Balanced creativity and coherence |
| `0.8` (default) | Moderately creative — suitable for roleplay dialogue |
| `0.9`–`1.5` | Highly creative, more varied output — can become less coherent |
| `1.5`–`2.0` | Maximum randomness — experimental use only |

## Storage

Model configs are stored as a JSON array under the key `neotavern_model_configs` in the storage layer. An additional key `neotavern_active_config_id` tracks which config is currently selected for chat.

```typescript
// Save a config
await settingsRepository.saveModelConfig(config);

// Retrieve all configs
const configs = await settingsRepository.getAllModelConfigs();

// Get active config
const activeId = await settingsRepository.getActiveConfigId();

// Set active config
await settingsRepository.setActiveConfigId(id);
```
