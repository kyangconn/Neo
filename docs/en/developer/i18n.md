# Internationalization

Internationalization (i18n) is handled by **react-i18next** with **i18next** as the core engine. Locale files use a namespace-based structure and are loaded eagerly at build time via `import.meta.glob`.

## Setup

The i18n system is initialized in `apps/desktop/src/i18n/index.ts`:

```typescript
import i18n from "i18next";
import { initReactI18next } from "react-i18next";

const resources: Record<string, Record<string, Record<string, unknown>>> = {};
const modules = import.meta.glob("../locales/*/*.json", { eager: true });

for (const [path, mod] of Object.entries(modules)) {
  const match = path.match(/locales\/([^/]+)\/([^/]+)\.json$/);
  if (!match) continue;
  const [, lang, ns] = match;
  if (!resources[lang]) resources[lang] = {};
  resources[lang][ns] = (mod as { default: Record<string, unknown> }).default;
}

i18n.use(initReactI18next).init({
  resources,
  lng: localStorage.getItem("neotavern_locale") || "zh",
  fallbackLng: "zh",
  defaultNS: "common",
  interpolation: { escapeValue: false },
  returnNull: false,
});
```

The initialization runs as a side effect import in `main.tsx`:

```typescript
import "./i18n"; // init i18next before first render
```

## Locale File Structure

Translation files live in `apps/desktop/src/locales/{lang}/{namespace}.json`:

```
src/locales/
├── en/
│   ├── common.json        # Shared UI labels
│   ├── toast.json         # Toast notification messages
│   ├── home.json          # Home page
│   ├── character.json     # Character management
│   ├── chat.json          # Chat interface
│   ├── preset.json        # Prompt presets
│   ├── worldbook.json     # World book
│   ├── neo-builder.json   # Character builder
│   ├── settings.json      # Settings
│   └── persona.json       # User persona
└── zh/
    ├── common.json
    ├── ...
```

### Namespaces

| Namespace | Content |
|-----------|---------|
| `common` | Shared UI: navigation, action labels (Save, Cancel, etc.) |
| `toast` | Toast/snackbar messages |
| `home` | Home page text |
| `character` | Character list, editor, import/export |
| `chat` | Chat UI, message actions, generation status |
| `preset` | Prompt preset management |
| `worldbook` | World book management, entry editor |
| `neo-builder` | Character builder wizard |
| `settings` | Settings pages (API, theme, regex, etc.) |
| `persona` | User persona editor |

### Loading Mechanism

All translation files are loaded eagerly at build time using `import.meta.glob("../locales/*/*.json", { eager: true })`. The glob matches `{lang}/{namespace}.json` paths, which are parsed and organized into the `resources` object keyed by `resources[lang][ns]`.

This means **no dynamic fetch at runtime** — all translations are bundled with the app. To add a new language, you only need to add locale JSON files under a new directory.

## Language Switching

The `changeLocale()` function switches the active language:

```typescript
import { changeLocale, getLocale, SUPPORTED_LOCALES } from "@/i18n";

// Switch to English
changeLocale("en");

// Get current locale
const current = getLocale(); // "en" | "zh"
```

It calls `i18next.changeLanguage()` and persists the choice to `localStorage` under the key `neotavern_locale`. On next app load, the stored locale is restored as the initial language.

## Usage Patterns

### Basic translation: `t()`

Use the `useTranslation` hook from react-i18next:

```tsx
import { useTranslation } from "react-i18next";

function MyComponent() {
  const { t } = useTranslation("common");

  return <button>{t("actions.save")}</button>;
}
```

The first argument to `useTranslation()` is the namespace. You can omit it when using `defaultNS`:

```tsx
const { t } = useTranslation(); // defaults to "common"
```

### Interpolation

```json
{
  "greeting": "Hello, {{name}}!"
}
```

```tsx
t("greeting", { name: "Alice" })
// → "Hello, Alice!"
```

### Nesting

```json
{
  "nav": {
    "home": "Home",
    "settings": "Settings"
  }
}
```

```tsx
t("nav.home") // → "Home"
```

## Adding a New Language

1. Create a new directory under `src/locales/` named with the language code (e.g. `ja/`).
2. Copy all namespace JSON files from an existing language (e.g. `en/`) as templates.
3. Translate each file's values — keep the keys intact.
4. Add the language code to the `SUPPORTED_LOCALES` array in `src/i18n/index.ts`:

```typescript
export const SUPPORTED_LOCALES = ["zh", "en", "ja"] as const;
export type Locale = (typeof SUPPORTED_LOCALES)[number];
```

5. Verify that `import.meta.glob` picks up the new files (it will automatically, since the glob pattern is `../locales/*/*.json`).

The `changeLocale()` function already accepts any string, so no additional wiring is needed. TypeScript will enforce the union type when using `Locale`.
