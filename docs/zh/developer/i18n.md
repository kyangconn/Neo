# 国际化

国际化（i18n）由 **react-i18next** 处理，底层引擎为 **i18next**。语言文件采用基于 namespace 的结构，并在构建时通过 `import.meta.glob` 热加载。

## 初始化

i18n 系统在 `apps/desktop/src/i18n/index.ts` 中初始化：

```typescript
import i18n from "i18next";
import { initReactI18next } from "react-i18next";

const resources: Record<string, Record<string, Record<string, unknown>>> = {};
const modules = import.meta.glob("../locales/*/*.json", { eager: true });

for (const [path, mod] of Object.entries(modules)) {
  const match = path.match(/locales\/([^/]+)\/([^/]+)\.json$/);
  if (!match) continue;
  const [, lang, ns] = match;
  if (!resources[lang]) resources[lang] = {};
  resources[lang][ns] = (mod as { default: Record<string, unknown> }).default;
}

i18n.use(initReactI18next).init({
  resources,
  lng: localStorage.getItem("neotavern_locale") || "zh",
  fallbackLng: "zh",
  defaultNS: "common",
  interpolation: { escapeValue: false },
  returnNull: false,
});
```

初始化通过副作用导入在 `main.tsx` 中执行：

```typescript
import "./i18n"; // 在首次渲染前初始化 i18next
```

## 语言文件结构

翻译文件位于 `apps/desktop/src/locales/{lang}/{namespace}.json`：

```
src/locales/
├── en/
│   ├── common.json        # 共享 UI 标签
│   ├── toast.json         # 通知消息
│   ├── home.json          # 首页
│   ├── character.json     # 角色管理
│   ├── chat.json          # 聊天界面
│   ├── preset.json        # 提示词预设
│   ├── worldbook.json     # 世界书
│   ├── neo-builder.json   # 角色构建器
│   ├── settings.json      # 设置
│   └── persona.json       # 用户人设
└── zh/
    ├── common.json
    ├── ...
```

### Namespace

| Namespace     | 内容                                    |
| ------------- | --------------------------------------- |
| `common`      | 共享 UI：导航、操作标签（保存、取消等） |
| `toast`       | 通知/提示消息                           |
| `home`        | 首页文本                                |
| `character`   | 角色列表、编辑器、导入/导出             |
| `chat`        | 聊天 UI、消息操作、生成状态             |
| `preset`      | 提示词预设管理                          |
| `worldbook`   | 世界书管理、条目编辑器                  |
| `neo-builder` | 角色构建向导                            |
| `settings`    | 设置页面（API、主题、正则等）           |
| `persona`     | 用户人设编辑器                          |

### 加载机制

所有翻译文件在构建时通过 `import.meta.glob("../locales/*/*.json", { eager: true })` 热加载。该 glob 匹配 `{lang}/{namespace}.json` 路径，解析后组织成以 `resources[lang][ns]` 为键的 `resources` 对象。

这意味着**运行时没有动态请求**——所有翻译都与应用打包在一起。要添加新语言，只需在新目录下添加语言 JSON 文件即可。

## 语言切换

`changeLocale()` 函数用于切换当前语言：

```typescript
import { changeLocale, getLocale, SUPPORTED_LOCALES } from "@/i18n";

// 切换到英语
changeLocale("en");

// 获取当前语言
const current = getLocale(); // "en" | "zh"
```

它会调用 `i18next.changeLanguage()`，并将选择持久化到 `localStorage` 中以 `neotavern_locale` 为键存储。下次加载应用时，会恢复已存储的语言作为初始语言。

## 使用模式

### 基础翻译：`t()`

使用 react-i18next 提供的 `useTranslation` hook：

```tsx
import { useTranslation } from "react-i18next";

function MyComponent() {
  const { t } = useTranslation("common");

  return <button>{t("actions.save")}</button>;
}
```

`useTranslation()` 的第一个参数是 namespace。使用 `defaultNS` 时可以省略：

```tsx
const { t } = useTranslation(); // 默认使用 "common"
```

### 插值

```json
{
  "greeting": "你好，{{name}}！"
}
```

```tsx
t("greeting", { name: "Alice" });
// → "你好，Alice！"
```

### 嵌套

```json
{
  "nav": {
    "home": "首页",
    "settings": "设置"
  }
}
```

```tsx
t("nav.home"); // → "首页"
```

## 添加新语言

1. 在 `src/locales/` 下创建一个以语言代码命名的新目录（例如 `ja/`）。
2. 从现有语言（例如 `en/`）复制所有 namespace JSON 文件作为模板。
3. 翻译每个文件的值——保持键名不变。
4. 将语言代码添加到 `src/i18n/index.ts` 的 `SUPPORTED_LOCALES` 数组中：

```typescript
export const SUPPORTED_LOCALES = ["zh", "en", "ja"] as const;
export type Locale = (typeof SUPPORTED_LOCALES)[number];
```

5. 验证 `import.meta.glob` 是否能捕获新文件（由于 glob 模式是 `../locales/*/*.json`，它会自动捕获）。

`changeLocale()` 函数已经接受任意字符串，因此无需额外配置。当使用 `Locale` 类型时，TypeScript 会强制约束联合类型。
