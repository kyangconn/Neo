# 预设

预设让你通过编写自定义指令（prompt）来自定义 **AI 的行为方式**，这些指令会随每条聊天消息一起发送。你可以把预设理解为 AI 本身的"性格卡片"——它们塑造助手的语气、规则和风格。

---

## 什么是预设？

每次你在聊天中发送消息时，Whale Play 都会构建一个 prompt，其中包含：

- 角色卡片
- 聊天历史
- 活跃的世界书条目
- 你的人设
- **预设卡片**——你编写的自定义指令

预设由一个或多个**卡片**组成。每张卡片都有一段文本，包含一个**角色**（system 或 user）和一个**顺序**。卡片按顺序组装后作为附加指令发送给 AI。

| 无预设 | 有预设 |
|---|---|
| AI 只知道角色和对话内容。 | AI 还会遵循你的自定义规则："始终以第三人称叙述"、"使用诗意的语言"、"禁止控物"。 |

---

## 创建预设

1. 在**左侧边栏**中，点击**预设图标** ⚡（预设）。
2. 点击 **"New Preset"**。
3. 为你的预设**命名**（例如"故事模式"）。
4. 开始向预设中添加卡片。

![预设页面](../images/presets-page.png)

## 向预设添加卡片

每张卡片是一条独立的指令。点击 **"Add Card"** 创建新卡片。

| 字段 | 是否必填 | 作用 | 示例 |
|---|---|---|---|
| **Name** | 是 | 卡片的标签，便于在列表中识别。 | 叙述风格 |
| **Role** | 是 | 该指令是作为 **system** 消息还是 **user** 消息发送。详见下方说明。 | system |
| **Content** | 是 | 实际的指令文本。 | 始终以生动的细节叙述动作和环境。运用所有五种感官。 |
| **Order** | 是 | 一个数字，决定该卡片相对于其他卡片的位置。数字小的在前。 | 1 |

### System 与 User 角色

| 角色 | 使用场景 |
|---|---|
| **System** | 最常用的选择。这些指令作为 AI 必须遵守的规则——类似于导演的提示。System 消息权重更高。 |
| **User** | 这些指令以你（用户）的口吻表达。适用于希望指令感觉像对话的一部分而非硬性规则的场景。 |

在实践中，你几乎都使用 **system**。仅在希望指令融入聊天历史时使用 **user**。

## 重新排序卡片

卡片按照你设置的顺序处理。要更改顺序：

- **拖放**卡片在列表中上下移动。顺序号会自动更新。
- 或手动编辑每张卡片的 **Order** 字段。

AI 按顺序接收卡片：编号最小的在前，编号最大的在后。

---

## 启用和禁用单张卡片

每张卡片都有一个**开关**。你可以打开或关闭卡片，而无需删除它们。

- **启用（开）：** 预设激活时包含该卡片。
- **禁用（关）：** 卡片被保存但被跳过。

这在你想临时移除某条指令而不丢失它时非常有用。例如，在和平场景中禁用"战斗规则"卡片，稍后再重新启用。

---

## 为聊天激活预设

预设按**角色**或按**聊天**设置。你可以为每个角色分配不同的预设。

1. 打开角色的**编辑对话框**（从角色页面）。
2. 找到 **Preset** 下拉选择器。
3. 选择要使用的预设。
4. 点击 **"Save"**。

此后，与该角色的每次聊天都会使用所选预设。如果未选择预设，则应用默认行为（无额外指令）。

## 导入和导出预设

你可以与他人分享预设，或将其备份到电脑上。

### 导出预设

1. 在预设页面上，找到要导出的预设。
2. 点击**导出（下载）图标**。
3. 一个 `.json` 文件将保存到你的电脑。

### 导入预设

1. 在预设页面上，点击 **"Import"**。
2. 从你的电脑中选择一个 `.json` 预设文件。
3. 该预设会添加到你的列表中。

导出的 JSON 结构如下：

```json
{
  "name": "故事模式",
  "cards": [
    {
      "name": "叙述风格",
      "role": "system",
      "content": "始终以生动的细节叙述动作和环境。",
      "order": 1,
      "enabled": true
    },
    {
      "name": "对话规则",
      "role": "system",
      "content": "对话必须用引号括起来。",
      "order": 2,
      "enabled": true
    }
  ]
}
```

> **📸 需要截图：** 打开卡片编辑器对话框，所有字段均已填写（Name、Role 下拉框显示"system"、Content 带有示例文本、Order 字段）。图片保存为 `docs/images/presets-card.png`。

## 常见用例的模板条目

以下是一些现成的卡片模板，你可以复制到自己的预设中。

### 生动故事讲述者

| 字段 | 值 |
|---|---|
| Role | system |
| Content | 用丰富的感官细节描述场景——视觉、听觉、嗅觉、触觉和味觉。使用文学语言，但保持对话自然且符合角色身份。 |

### 战斗规则

| 字段 | 值 |
|---|---|
| Role | system |
| Content | 战斗采用回合制。生动地描述攻击及其后果。用户的操作可以根据故事情境成功或失败。不得控制用户的角色或决定其行动的结果。 |

### 极简 / 快节奏

| 字段 | 值 |
|---|---|
| Role | system |
| Content | 保持回复简短有力。不要华丽的描写。使用快速的对话和最少的叙述。 |

### 禁止控物（公平游戏）

| 字段 | 值 |
|---|---|
| Role | system |
| Content | 你只能控制你自己的角色。不得替用户角色行动，不得决定用户角色行动的结果，不得指定用户角色的反应或情绪。 |

### 第三人称叙述

| 字段 | 值 |
|---|---|
| Role | system |
| Content | 全部以第三人称叙述。用角色名称或"ta"来指代你的角色。 |
