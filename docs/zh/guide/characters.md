# 角色

角色是 Whale Play 的核心。每个角色都是一个数字化的"人格"，AI 在聊天中将扮演该角色。本指南涵盖了创建、编辑、导入和导出角色所需的一切知识。

---

## 角色列表

打开角色页面时，你会看到所有已创建角色的列表。每个条目显示角色的**名称**、**头像**和简短**描述**。从这里你可以点击角色开始聊天，或使用操作按钮进行编辑或删除。

## 创建角色

点击 **"New Character"** 打开创建对话框。填写以下字段——你提供的信息越详细，AI 的回复就越丰富。

| 字段                  | 是否必填 | 作用                                                                                   | 示例                                                                                        |
| --------------------- | -------- | -------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| **Name**              | 是       | 角色名称。AI 会以此名称自称。                                                          | 露娜                                                                                        |
| **Description**       | 否       | 角色的简短介绍。帮助 AI 快速了解角色。                                                 | 一位守护古老图书馆的神秘森林精灵                                                            |
| **Personality**       | 否       | 定义角色行为方式的关键特质。以逗号分隔的列表形式填写。                                 | 智慧、好奇、温柔、俏皮、调皮                                                                |
| **Scenario**          | 否       | 角色所处的设定或情境。为对话搭建舞台。                                                 | 你发现自己身处一片隐秘的林间空地，一棵巨大的柳树根系中生长着一座古老的石头图书馆。          |
| **First Message**     | 是       | 开始新聊天时角色发出的第一条消息。这为整个对话定下基调。                               | _"欢迎你，旅人。我已等候多时。这些书几天来一直在低语你的名字。"_                            |
| **Example Dialogues** | 否       | 展示角色说话方式的简短示例对话。帮助 AI 匹配角色的语言风格。每条示例都是一段来回对话。 | `你：你是谁？` `露娜：我是这片森林的守护者。这里的每一本书都承载着一段记忆——其中也有你的。` |

![角色编辑对话框](../images/characters-edit.png)

> **📸 需要截图：** 打开"New Character"对话框，填写所有字段（使用上方示例数据）。截取整个对话框表单。图片保存为 `docs/images/characters-edit.png`。

### 打造优秀角色的小贴士

- **First Message 最重要。** AI 用它来建立角色的语气、设定和声线。使其富有描述性并符合角色身份。
- **Personality 特质** 应简短有力。可以理解为标签：`勇敢、笨拙、戏剧化`。
- **Scenario** 类似于舞台说明——告诉 AI 对话发生的地点和背景。
- **Example Dialogues** 非常有效。展示 2–3 组对话，演示角色对不同情境的反应。

## 编辑角色

1. 进入**角色**页面。
2. 找到你想要编辑的角色。
3. 点击其名称旁边的**编辑（铅笔）图标**。
4. 同样的创建对话框会打开，所有字段已预填。
5. 进行修改后点击 **"Save"**。

修改会立即生效——此后与该角色的新聊天将使用更新后的卡片。已有聊天不受影响。

---

## 删除角色

1. 进入**角色**页面。
2. 找到你想要删除的角色。
3. 点击**删除（垃圾桶）图标**。
4. 在弹出的对话框中确认删除。

> **⚠️ 警告：** 删除角色也会删除该角色的所有聊天记录和存档点。此操作不可撤销。

## 导入角色卡片

Whale Play 支持**角色卡片格式**——一种打包 AI 角色扮演角色数据的标准方式。你可以从 `.json` 或 `.png` 文件导入卡片。

### 支持的文件格式

| 格式      | 工作原理                                                                                                  |
| --------- | --------------------------------------------------------------------------------------------------------- |
| **.json** | 包含角色卡片数据的纯 JSON 文件。这是标准的卡片导出格式。                                                  |
| **.png**  | 角色卡片数据**嵌入**在文件元数据中的 PNG 图片。图片本身可作为角色的头像。这类文件通常称为"角色卡片 PNG"。 |

### 如何导入

1. 进入**角色**页面。
2. 点击 **"Import"** 按钮（或上传图标）。
3. 从你的电脑中选择一个 `.json` 或 `.png` 文件。
4. 角色会自动添加到你的列表中。

如果导入的是包含嵌入卡片数据的 `.png`，该图片将用作角色头像。如果导入的是 `.json`，你可以另外设置头像。

### 在哪里寻找角色

许多角色卡片的创作者会在网上分享他们的卡片。寻找以 `.card.json` 结尾的文件或角色卡片 `.png` 图片。[Character Hub](https://chub.ai) 等社区以及各种 Discord 服务器上托管着成千上万个现成的角色。

---

## 导出角色

你可以将任何角色导出为 `.json` 文件，以便与他人分享、备份或在设备间迁移。

1. 进入**角色**页面。
2. 找到你想要导出的角色。
3. 点击其名称旁边的**导出（下载）图标**。
4. 一个 `.json` 文件将保存到你的电脑。

导出的文件使用标准角色卡片格式，因此也可以导入到其他兼容的应用中。

---

## 角色卡片结构

以下是角色卡片内部的结构，以及每个字段如何影响 AI 的行为：

```json
{
  "name": "露娜",
  "description": "一位守护古老图书馆的神秘森林精灵",
  "personality": "智慧、好奇、温柔、俏皮",
  "scenario": "你发现自己身处一片隐秘的林间空地……",
  "first_mes": "*欢迎你，旅人。*",
  "mes_example": "你：你是谁？\n露娜：我是这片森林的守护者。",
  "avatar": "base64 编码的图片数据",
  "system_prompt": ""
}
```

| 字段            | 对 AI 的影响                                        |
| --------------- | --------------------------------------------------- |
| `name`          | 角色的身份标识。AI 会以此名称自称。                 |
| `description`   | 添加到 AI 的 system prompt 中，作为角色的简短介绍。 |
| `personality`   | 作为特质列表注入到 prompt 中。指导 AI 的行为风格。  |
| `scenario`      | 描述当前情境。AI 用此来理解设定和上下文。           |
| `first_mes`     | 开场消息。设定对话的语气、声线和起始点。            |
| `mes_example`   | Few-shot 示例，展示期望的对话模式和回复风格。       |
| `avatar`        | 可选图片，显示为角色的图标。                        |
| `system_prompt` | 覆盖该角色的默认 system prompt。不熟悉时请留空。    |
