# 世界书

世界书是一系列传说、事实和背景信息的集合，AI 可以在聊天过程中从中获取信息。你可以把它理解为**角色世界的维基百科**——AI 会读取相关条目，从而使回复更加一致且富有沉浸感。

---

## 什么是世界书？

你是否曾在一次精彩的角色扮演中，却发现 AI 忘记了设定的某个重要细节？这就是世界书发挥作用的地方。

世界书是一个**条目数据库**。每个条目包含一条信息——一个地点、一个角色、一个事件、一件物品或一条规则。当你聊天时，Whale Play 会根据世界书检查你的消息。如果你说的内容匹配了某个条目，AI 在回复之前会先读取该条目。

例如：

| 条目     | 内容                                                           |
| -------- | -------------------------------------------------------------- |
| 月石     | 一块魔法蓝宝石，附近有人说谎时会发出光芒。藏于回音之井中。     |
| 奥林船长 | 饱经风霜的老水手，装有一条机械腿。知道穿越珊瑚海峡的秘密通道。 |
| 沉默法则 | 在维里迪亚城，午夜后说话为古老法令所禁止。                     |

如果你在聊天中提到"月石"或"发光的蓝宝石"，AI 就会确切地知道你在说什么——即使是在几个小时前提到的。

## 创建世界书

1. 在**左侧边栏**中，点击**书籍图标** 📖（世界书）。
2. 点击 **"New World Book"**。
3. 为其**命名**（例如"维里迪亚王国"）。
4. 可选：添加简短**描述**说明这本世界书涵盖的内容。

![世界书编辑器](../images/worldbook-editor.png)

> **📸 需要截图：** 打开世界书页面，显示 1–2 本世界书的列表，同时"New World Book"按钮可见。图片保存为 `docs/images/worldbook-editor.png`。

## 添加条目

创建世界书后，你可以向其中添加条目。每个条目是一条信息。

1. 从列表中打开一本世界书。
2. 点击 **"Add Entry"**。
3. 填写条目字段：

| 字段             | 是否必填 | 作用                                                                               | 示例                                                                               |
| ---------------- | -------- | ---------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------- |
| **Title**        | 是       | 该条目的简短名称。用于显示。                                                       | 月石                                                                               |
| **Keywords**     | 是       | 触发该条目的词语或短语。当聊天中出现这些词时，条目会被注入。多个关键词用逗号分隔。 | 月石、蓝宝石、发光宝石、谎言                                                       |
| **Content**      | 是       | 实际的传说文本。这是 AI 将要读取的内容。                                           | 一块魔法蓝宝石，附近有人说谎时会发出光芒。它在数百年前被藏于王宫下方的回音之井中。 |
| **Priority**     | 否       | 一个数字（0–100）。如果多个条目同时被触发，优先级高的条目先被注入。默认为 0。      | 10                                                                                 |
| **Trigger Mode** | 是       | 该条目的使用方式——见下方说明。                                                     | Trigger                                                                            |

### 条目类型：Always 与 Trigger

这是每个条目最重要的设置。

| 类型        | 图标 | 行为                                                                                             |
| ----------- | ---- | ------------------------------------------------------------------------------------------------ |
| **Always**  | 📌   | 该条目**始终**包含在每次聊天的 prompt 中，无论对话内容是什么。用于 AI 绝不能忘记的核心世界信息。 |
| **Trigger** | 🔍   | 仅当某个**关键词**出现在最近的对话中时，该条目才会被包含。用于并非始终相关的情境细节。           |

**何时使用 Always：**

- 世界的核心前提（例如"魔法需要血的代价"）
- 影响所有对话的规则（例如"角色不能说谎"）
- 用户当前所处的关键地点

**何时使用 Trigger：**

- 可能出现也可能不出现的特定 NPC
- 关于遥远地点的传说
- 历史事件和背景故事
- 物品描述

## 条目如何注入 Prompt

当 AI 准备回复时，它会从多个部分构建 prompt。世界书条目根据其**位置**设置被插入。

| 位置               | 插入位置                                                        | 最佳用途                                     |
| ------------------ | --------------------------------------------------------------- | -------------------------------------------- |
| **Before History** | 添加在聊天历史（迄今为止的对话）之前。AI 将其视为"背景上下文"。 | 世界规则、设定描述、角色动机。               |
| **After History**  | 添加在聊天历史之后、AI 回复之前。AI 将其视为"即时上下文"。      | 近期事件、用户刚问及的特定物品、时效性信息。 |

在实践中，**Before History** 是默认选项，适用于大多数条目。仅在需要信息感觉新鲜且即时时使用 **After History**。

## 激活与停用世界书

你可以拥有多本世界书，但只有**激活的**世界书才会在聊天中被检查。

- **开关切换：** 使用列表中每本世界书名称旁的开关。
- **激活：** 该世界书的条目会与每条消息进行匹配检查。
- **停用：** 世界书会被保存，但在聊天中被忽略。

![条目编辑面板](../images/worldbook-entry.png)

> **📸 需要截图：** 打开条目的编辑面板，显示所有已填写的字段（Title、Keywords、Content、Priority、Trigger Mode 选择器和 Position 选择器）。图片保存为 `docs/images/worldbook-entry.png`。

### 世界书使用技巧

- **从小处着手。** 一本拥有 5–10 个精心编写条目的世界书，比一本有 50 条散乱笔记的世界书更有效。
- **关键词至关重要。** 思考用户自然会说哪些词来引用这条信息。包含同义词和变体。
- **内容保持简洁。** 每条条目 2–5 句话即可。AI 的上下文空间有限。
- **合理使用优先级。** 优先级为 100 意味着触发时该条目几乎总会出现。将其留给关键传说。
- **避免重复。** 如果两个条目可能因同一关键词被触发，AI 可能会混淆。
