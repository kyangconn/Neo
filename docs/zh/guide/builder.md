# Whale Builder

## 什么是 Whale Builder？

Whale Builder 是 Whale Play 内置的一种**对话驱动的角色创建助手**。无需填写表单或编辑 JSON，你只需用自然语言描述你的角色构思，构建器就会引导你通过结构化的流程，生成一个完整且可用的角色卡片。

构建器由**技能系统**驱动——专门的 AI 角色能够精确地打造角色、编写世界书传说并组装最终的角色卡片。每一步都受到引导，过程透明且协作。

**最终产出：**
- 一张格式完整的**角色卡片**（人设、特质、技能、背景故事）
- 一份**性格调色板**（情感范围和行为特质）
- **世界书条目**（与角色相关的传说、地点、NPC、物品）
- 一份**创建计划**（构建器遵循的结构化蓝图）
- 一份**评估报告**（质量检查与改进建议）

## 开始使用

你可以从两个位置启动构建器：

1. **角色页面** — 点击 **"Whale Builder"** 按钮。
2. **直接 URL** — 导航到 `/character-builder`。

启动后，只需描述你的角色构思。例如：

> *"我想要一个 rogue 风格的考古学家，探索古代遗迹，使用鞭子，并且有一个总比他快一步的教授对手。"*

构建器会接手后续工作。

## 创建流程

构建器遵循结构化的 6 步流程。每一步都建立在前一步的基础上。

### 1. 收集方向

告诉构建器你的角色概念。你可以描述得非常详细，也可以非常随意。构建器会提出澄清性问题来完善构思——比如设定、性格、目标和关键关系。

构建器可能会调用 `ask_user_options`，一次性向你提出 2–5 个结构化问题，让你一次性回答多个选择。

### 2. 对齐计划

一旦构建器理解了你的愿景，它会提出一个**创建计划**——一份结构化的 YAML 文档，概述：

- 角色类型和定位
- 需要生成的关键特质和技能
- 需要的世界书条目（地点、势力、物品）
- 叙事钩子和背景故事节拍

你可以查看计划、提出修改建议，或批准它以便继续。

**示例制品 —— `creation-plan.yaml`：**

```yaml
character:
  name: 艾拉·维克斯
  archetype: Rogue 考古学家
  traits: [好奇、狡猾、孤独]
  skills: { 考古学: 4, 潜行: 3, 近战: 2 }
worldbook:
  - "奥莎沉没神庙"
  - "教授的日记"
  - "青铜钥匙教团"
narrative_hooks:
  - "对手已经率先找到了神庙"
  - "神器被诅咒了"
```

### 3. 搜索参考资料（可选）

如果你启用**网络搜索**，构建器可以为真实性查找现实世界的参考资料——历史文物、真实地点、神话或符合时代背景的细节。

当你需要基于事实的世界构建时，打开此功能。构建器会使用 `web_search` 工具查找相关信息。

### 4. 生成条目

构建器逐一创建**世界书条目**。这些是你的角色所生活的传说基石：

- 地点（神庙、城市、遗迹）
- 势力和组织
- 关键 NPC（盟友、对手、导师）
- 物品和神器
- 历史事件

每个条目的生成都会参考之前的条目，确保世界保持一致。

### 5. 生成角色卡片

世界构建完毕后，构建器会组装**最终的角色卡片**——一个完整的、可用于 Whale Play 的角色。其中包括：

- 姓名、外貌和性格
- 特质和技能评级
- 背景故事和动机
- 背包和起始装备
- 与世界书条目的关系图

构建器会使用 `validate_character_draft` 检查卡片的完整性和一致性，然后呈现给你。

### 6. 保存到 Whale Play

一旦你满意，构建器会调用 `save_character_draft` 来保存角色。这是**唯一**让你的角色在 Whale Play 中可用的操作。

保存后，你可以立即与你的新角色开始聊天。

## 制品

在构建器执行流程的过程中，它会生成多个制品。这些制品会显示在右侧面板中，方便你随时查看。

### `creation-plan.yaml`

指导生成的结构化蓝图。你可以在条目创建前查看并修改它。

### 性格调色板

角色情感范围和行为特质的可视化展示。显示角色在压力下、社交场合以及追求目标时的反应方式。

### 角色卡片

最终输出——一个完整的、可用于 Whale Play 的角色。包含特质、技能、背景故事、背包和关系。

### 世界书

生成的传说条目，为你的角色世界增添深度。每个条目都是一个自包含的片段，聊天系统可以将其注入对话中以获得更丰富的角色扮演体验。

### 评估报告

角色卡片创建后生成的质量评估报告。它会检查：

- **完整性**——所有必填字段是否已填写？
- **一致性**——特质、技能和背景故事是否协调一致？
- **表现力**——角色是否有足够的人格深度？
- **建议**——构建器推荐的具体改进

你可以在修改后请求重新评估，查看分数是否有所提升。

## 技巧

- **角色概念要具体**——你一开始提供的细节越多，构建器所需澄清的轮次就越少。
- **如果需要历史设定、文物或神话的真实世界参考资料，请启用网络搜索。**
- **仔细审查创建计划**——它是后续所有工作的蓝图。在此阶段进行修改成本较低。
- **使用"Evaluate Draft"** 在最终定稿前获取改进建议。评估报告会指出你可能遗漏的缺口。
- **每个世界书条目都在上下文中生成**——早期条目会影响后续条目，因此请逐一审查。
- **你可以迭代**——如果结果不理想，描述你想要修改的内容，构建器可以重新生成特定部分。

## 底层实现

构建器采用**基于技能（Skill）的架构**。当你启动一个会话时，系统会向 AI 的 system prompt 中注入一个 Skill 指令，定义：

1. 完整的工作流程（上面的 6 个步骤）
2. 数据格式（YAML 计划结构、角色卡片模式、世界书条目格式）
3. 写作规则（语气、深度、一致性指南）
4. 工具使用（何时搜索、何时提问、何时保存）

Skill 参考文件位于 `apps/desktop/src/features/character/neo-builder-skill-references/`，通过 `read_skill_reference` 和 `list_skill_references` 工具按需加载。

这种设计意味着可以通过编辑 Skill 文件来更新构建器的行为，而无需修改代码——使其灵活且易于持续改进。
